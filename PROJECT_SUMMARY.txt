              MIWESU FIRE WOOD - NEXT.JS PROJECT SUMMARY

PROJECT TYPE
------------
Next.js App Router application for Miwesu Fire Wood: a marketing and
e-commerce-style site for premium firewood and braai wood delivery in Gauteng.
No backend or database; content is driven by TypeScript/JSON data (products,
woods, suburbs, site config). Ordering flows to WhatsApp; no online checkout.

--------------------------------------------------------------------------------
NEXT.JS VERSION & STACK
--------------------------------------------------------------------------------

• Next.js: 15.x (^15.1.0 in package.json; typically 15.5.x when installed)
• React: 19.x
• TypeScript: 5.7.x
• Styling: Tailwind CSS 3.4.x
• Font: Inter (Google Font, via next/font)
• Key dependencies: lucide-react (icons), html-to-image & downloadjs (admin
  ad generator export)

Scripts:
  npm run dev   - Development with Turbopack (next dev --turbopack)
  npm run build - Production build
  npm run start - Production server
  npm run lint - ESLint

LIST OF ALL PAGES (ROUTES)

PUBLIC PAGES
------------

/ (Home)
  Homepage. Hero, value props, wood finder, three product sections (Sekelbos,
  Geelhaak, Braai Mix) with 3 products each, contact form. Optional ?product=id
  pre-selects product in form. Dynamic (server-rendered).

/products
  All products. Lists all 9 products in three sections (Sekelbos, Geelhaak,
  Braai Mix). Links to /products/sekelbos, /products/geelhaak, /products/braai-mix
  and to individual product pages. Static.

/products/[id]
  Product detail. [id] = geelhaak-10, braai-mix-10, sekelbos-10, geelhaak-20,
  braai-mix-20, sekelbos-20, geelhaak-30, braai-mix-30, sekelbos-30. Image
  gallery, description, price, MOQ, CTA to WhatsApp. ProductSchema (with
  AggregateRating + Review when reviews exist), FAQPageSchema. SSG via generateStaticParams.

/products/sekelbos
  Sekelbos category. Copy for Sekelbos wood, three products (10/20/30 kg). Static.

/products/geelhaak
  Geelhaak category. Copy for Geelhaak wood, three products (10/20/30 kg). Static.

/products/braai-mix
  Braai Mix category. Copy for Braai Mix, three products (10/20/30 kg). Static.

/woods
  "Meet our woods" index. Educational list of wood species (Geelhaak,
  Knoppiesdoring, Sekelbos, Snuifpeul, Mopane, Rooibos). Links to each
  /woods/[slug]. Static.

/woods/[slug]
  Wood species page. [slug] = geelhaak, knoppiesdoring, sekelbos, snuifpeul,
  mopane, rooibos. Rich content: intro, appearance (bark, heartwood, sapwood),
  burn profile (heat, coals, smoke, lighting), best uses, "In our range" note.
  Hero and gallery images. HowTo schema. SSG via generateStaticParams.

/delivery
  Delivery policy. Text policy: free delivery conditions, zones, MOQ, curbside,
  access, redelivery. FAQPageSchema (delivery variant). Static.

/delivery-areas
  Delivery areas index. List of Gauteng suburbs/areas with links to
  /delivery/[suburb]. Static.

/delivery/[suburb]
  Suburb-specific delivery landing. 47 suburbs (see Data Sources). Zone (A/B/C),
  tagline, high-intent meta (next-day, kiln-dried, sub-12% moisture), CTA.
  Requests for excluded township slugs (e.g. /delivery/soweto) return 404.
  SSG via generateStaticParams.

/policies
  Policies index. Links to Delivery, Terms, Returns, Product & Wood policy. Static.

/terms
  Terms and conditions. Legal terms of sale/supply. Static.

/returns
  Returns and refunds policy. Static.

/product
  Product & Wood policy. Nature of wood, moisture, grading, defects. Static.

SPECIAL / SYSTEM
----------------

/robots.txt
  Generated route. Allow all; sitemap URL. (app/robots.ts)

/sitemap.xml
  Generated sitemap. Home, products, product by id, woods, wood by slug,
  delivery-areas, all 47 delivery suburb URLs, policies, terms, returns,
  delivery, product policy. Dynamic from lib data. (app/sitemap.ts)

/admin/ad-generator
  Admin tool. Client-side. Create ad creatives (vertical, square, horizontal):
  heading, subheading, specs, phone, neighborhood, background image, then
  export as PNG. Uses ad-presets and neighborhoods data. Not for end customers.

DATA SOURCES (LIB)
--------------------------------------------------------------------------------

• lib/products.ts     - PRODUCTS array (9 products), getProductById. Prices, MOQ, copy, image paths.
• lib/wood-types.ts   - WOOD_TYPES (sekelbos, geelhaak, braai-mix), productIds per type, getProductsByWood.
• lib/woods.ts        - WOODS (species: geelhaak, knoppiesdoring, sekelbos, snuifpeul, mopane, rooibos). Educational content only.
• lib/suburbs.ts      - SUBURBS (47 Gauteng areas). slug, name, areaName, zone (A/B/C), tagline. Used for programmatic delivery pages and LocalBusiness areaServed.
• lib/excluded-suburbs.ts - EXCLUDED_SUBURB_SLUGS (townships/informal settlements). isExcludedSuburbSlug(). Visiting /delivery/[excluded] returns 404.
• lib/reviews.ts      - Static customer reviews per product for Product schema (AggregateRating + Review). getReviewsForProduct, getAggregateRatingForProduct.
• lib/site.ts         - SITE_URL, SITE_NAME, SITE_DESCRIPTION, WHATSAPP_NUMBER, WHATSAPP_DISPLAY, SITE_PHONE, WHATSAPP_ORDER_LINK, SITE_KEYWORDS, SITE_FACEBOOK_URL, SITE_SAME_AS.
• lib/ad-presets.ts   - Ad generator presets (e.g. Fire Special).
• lib/neighborhoods.ts - Neighborhoods for ad generator (e.g. Meyersdal, Sandton).

--------------------------------------------------------------------------------
SEO: WHAT'S BEEN DONE AND HOW
--------------------------------------------------------------------------------

Technical foundation
--------------------
• SSG for all listable routes: generateStaticParams used for /products/[id], /woods/[slug], /delivery/[suburb]. Pre-rendered at build time for fast TTFB and LCP; maximum crawl budget efficiency.
• metadataBase set to SITE_URL in app/layout.tsx so all relative OG images and canonicals resolve correctly.
• title.template in layout: "%s | Miwesu Fire Wood" so every page gets consistent brand suffix.
• generateMetadata on dynamic routes (products, woods, delivery) for unique title, description, keywords, openGraph, twitter, alternates.canonical per URL. No trailing slash in config; sitemap URLs match.

Structured data (JSON-LD)
-------------------------
• LocalBusinessSchema (root layout)
  - @type HomeGoodsStore. areaServed: explicit array of City objects from SUBURBS only (no generic "Gauteng"), so search engines understand exact service area.
  - contactPoint: WhatsApp as primary (telephone, url wa.me, contactType, option "WhatsApp ordering").
  - sameAs: Facebook and any other verified profiles. telephone: SITE_PHONE (WhatsApp display number).
  - description includes "Kiln-verified sub-12% moisture", "Order via WhatsApp".

• ProductSchema (every /products/[id])
  - Product with name, description, image, brand, sku, offers (url, priceCurrency ZAR, price, availability InStock).
  - additionalProperty: Minimum order, Delivery, Moisture content, Verified moisture.
  - When lib/reviews.ts has reviews for that product: aggregateRating (ratingValue, reviewCount, bestRating 5, worstRating 1) and review[] (author, datePublished, reviewBody, reviewRating). Enables star ratings in SERPs and higher CTR.

• FAQPageSchema (components/json-ld/FAQPageSchema.tsx)
  - Two variants. "delivery": used on /delivery (policy). "product": used on each /products/[id].
  - delivery: Estate delivery (Waterfall, Dainfern), moisture guarantee, next-day delivery, closed combustion suitability.
  - product: Best hardwood for closed combustion, pizza ovens, minimum order.
  - Outputs FAQPage with mainEntity[]. Question/Answer. Targets PAA (People Also Ask) and Position Zero.

• BreadcrumbListSchema on policy and product/delivery/woods pages for clearer SERP breadcrumbs.

• HowToWoodSchema on /woods/[slug] for step-by-step informational queries (existing).

Geospatial (programmatic + negative)
------------------------------------
• Programmatic delivery pages: 47 affluent Gauteng suburbs in lib/suburbs.ts. Each gets a unique /delivery/[suburb] page with zone, tagline, and high-intent meta. Sitemap includes all 47; generateStaticParams builds only these (no excluded slugs).
• Negative SEO / exclusion: lib/excluded-suburbs.ts defines EXCLUDED_SUBURB_SLUGS (e.g. soweto, alexandra, mamelodi, tembisa, kagiso). app/delivery/[suburb]/page.tsx calls isExcludedSuburbSlug(suburb) in generateMetadata and in the page component; if true, returns 404 (notFound()). No landing pages or crawl budget spent on townships or informal settlements.

High-intent keywords and meta
-----------------------------
• Site-wide (lib/site.ts): SITE_DESCRIPTION and SITE_KEYWORDS include "kiln-verified sub-12% moisture", "next-day firewood delivery Gauteng", "best hardwood closed combustion stoves South Africa", "firewood stacking service Johannesburg", etc.
• Delivery policy (/delivery): title/description mention next-day, kiln-dried, sub-12%, estate delivery. FAQ schema backs this.
• Delivery suburb pages: title includes "Next-Day Braai Wood"; description includes "Kiln-dried, sub-12% moisture", "Order via WhatsApp"; keywords include "next-day firewood delivery", "sub-12% moisture firewood".
• Product pages: description includes "Sub-12% moisture, kiln-verified"; keywords include "sub-12% moisture firewood", "closed combustion firewood".
• Suburb taglines in lib/suburbs.ts use "sub-12% moisture", "next-day", "closed combustion", "kiln-verified" where relevant.

Crawl and indexing
------------------
• app/sitemap.ts dynamically builds sitemap from PRODUCTS, WOODS, getAllSuburbSlugs() and static routes. No trailing slashes; matches Next.js default (trailingSlash not set).
• app/robots.ts allows all and points to sitemap. No disallow for important paths.
